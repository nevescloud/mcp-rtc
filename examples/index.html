mcp-rtc

GitHub SPEC npm

Turn any browser tab into a Claude-callable MCP server. No Node bridge, no install, no public URL.

@jonasneves/mcp-rtc-bridge-tab takes a remote MCP server reachable over WebRTC and re-exposes its tools as WebMCP tools in the local browser tab. Local Claude (Code, Desktop, claude.ai with the Anthropic Chrome extension) can then call them natively. The spec and reference transport exist to make that work and let other implementations interoperate.

Try it — hello-tool

1. Open a browser tab as an MCP server.

Exposes one tool, get_greeting, that returns a greeting with a random adjective + noun (so the call is verifiable as having reached the tab).

Open hello.html →

2a. Call it from Claude Code (Path B).

Add to ~/.claude/settings.json:

{ "mcpServers": { "hello": { "command": "npx", "args": ["-y", "@jonasneves/mcp-webrtc-bridge", "connect", "<site-id>"] } } }

Site id appears in hello.html's URL bar (?site=hi-XXXXXX). Then ask Claude "use the hello server's get_greeting tool."

2b. Call it from Claude.ai / Claude Desktop (Path C — the headline).

Open the bridge tab — requires Chrome 146+ with the Anthropic Claude extension installed. The bridge re-exposes get_greeting via navigator.modelContext; any local Claude can then call it.
