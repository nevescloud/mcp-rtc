mcp-rtc

GitHub SPEC npm

Turn any browser tab into a Claude-callable MCP server.

No Node bridge, no install, no public URL. Local Claude (Code, Desktop, claude.ai with the Anthropic Chrome extension) calls a remote tab's tools natively, over a WebRTC data channel.

Try it · two tabs, one machine

★ headline pattern · Path C

Browser tab calls another browser tab's tool, via local Claude.

  1. Open hello.html. The tab becomes an MCP server with one tool, get_greeting. Note the ?site=hi-XXXXXX the URL generates.
  2. Open neves.cloud/c/#site=hi-XXXXXX in a tab where you have the Anthropic Claude Chrome extension installed.
  3. Ask Claude "call get_greeting". The call routes peer-to-peer over WebRTC; the result comes back through WebMCP.
Open hello.html → Open /c/ →

Requires: Chrome 146+ with the Anthropic Claude extension installed.

Or call it from Claude Code (Path B).

Same hello.html, different consumer. Add to ~/.claude/settings.json:

{ "mcpServers": { "hello": { "command": "npx",
  "args": ["-y", "@jonasneves/mcp-webrtc-bridge",
           "connect", "<site-id>"] } } }

Then ask Claude "use the hello server's get_greeting tool." Works against any mcp-rtc server, not just hello.html.
