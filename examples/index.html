Turn any browser tab into a Claude-callable MCP server.

No Node bridge, no install, no public URL. The protocol — WebMCP↔WebRTC — carries tool calls peer-to-peer between a tab and any local Claude.

    tab (the MCP server)            mcp-rtc             local Claude
    ────────────────────         (WebRTC data            ────────────
    register tools via            channel,
    @jonasneves/mcp-rtc           peer-to-peer)
                       │                                ┌─► Claude Code  via stdio bridge
                       └──────────────────────────────► │   (npx mcp-webrtc-bridge)
                                                        │
                                                        └─► Claude.ai    via bridge tab + WebMCP
                                                            (Chrome 146+ with consumer)

Try it · two tabs, one machine

★ headline pattern · Path C

Browser tab calls another browser tab's tool, via local Claude.

  1. Open neves.cloud/h/. The tab becomes an MCP server with one tool, get_greeting. Note the ?site=hi-XXXXXX the URL generates.
  2. Open neves.cloud/b/#hi-XXXXXX in a Chrome 146+ tab with a WebMCP consumer (e.g. the Anthropic Claude extension).
  3. Ask Claude "call get_greeting". The call routes peer-to-peer over WebRTC; the result comes back through WebMCP.
Open /h/ → Open /b/ →

Requires: Chrome 146+ with a WebMCP consumer (e.g. the Anthropic Claude extension). Only for this path — Path B below uses no browser.

Or use Claude Code (Path B). No browser needed.

  1. Add to ~/.claude/settings.json and restart: 
    {
  "mcpServers": {
    "mcp-webrtc-bridge": {
      "command": "npx",
      "args": ["-y", "@jonasneves/mcp-webrtc-bridge"]
    }
  }
}
  2. Open neves.cloud/h/ and copy the "Paste in Claude Code" prompt — it looks like Connect neves.cloud/b#hi-XXXXXX.
  3. Paste it into Claude Code. Claude calls connect with the site id, lists the remote tools, then asks what to do.

Works against any mcp-rtc server, not just hello.html. Same flow for Cursor / Claude Desktop with their MCP-server config.
