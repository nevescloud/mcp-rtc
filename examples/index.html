Turn any browser tab into a Claude-callable MCP server.

No Node bridge, no install, no public URL. The protocol carries tool calls peer-to-peer between a tab and any local Claude — over WebMCP and WebRTC.

Tab The MCP server. @nevescloud/mcp-rtc
→
mcp-rtc WebRTC data channel, peer-to-peer.
→
Claude Code via stdio bridge npx @nevescloud/mcp-rtc-bridge
Claude.ai via bridge tab + WebMCP Chrome 146+ with consumer

Try it

Browser ↔ browser, via local Claude.

  1. Open neves.cloud/h/. The tab becomes an MCP server with one tool, get_greeting. Note the #hi-XXXXXX the URL generates.
  2. Open neves.cloud/b/#hi-XXXXXX in a Chrome 146+ tab with a WebMCP consumer (e.g. the Anthropic Claude extension).
  3. Ask Claude "call get_greeting". The call routes peer-to-peer over WebRTC; the result comes back through WebMCP.
Open the example tab → Open the bridge tab →

Requires: Chrome 146+ with a WebMCP consumer (e.g. the Anthropic Claude extension).

webmcp webrtc bridge-tab no-install

Browser ↔ Claude Code, from your terminal.

  1. Add to ~/.claude/settings.json and restart: 
    {
  "mcpServers": {
    "mcp-rtc-bridge": {
      "command": "npx",
      "args": ["-y", "@nevescloud/mcp-rtc-bridge"]
    }
  }
}
  2. Open neves.cloud/h/ and copy the "Paste in Claude Code" prompt — it looks like Connect neves.cloud/b#hi-XXXXXX.
  3. Paste it into Claude Code. Claude calls connect with the site id, lists the remote tools, then asks what to do.

Works against any mcp-rtc server, not just hello.html. Same flow for Cursor / Claude Desktop with their MCP-server config.

stdio mcp claude code no browser
